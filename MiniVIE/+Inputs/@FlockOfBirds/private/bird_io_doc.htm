Overview

Matlab Version 6 provides support of serial port communication. This document describes Matlab script files which acquire data from a flock of birds. Advantages of using Matlab for bird I/O are:

  • Easy to program language with increase user interface and GUI support
  • Easy to generate 3D plotting functions
  • Interpretive language provides quick debugging

Matlab does run slower than complied languages such as C, but generally results have been satisfactory.

The included script files have been written to allow quick setup and easy customization. I have tried to strip down the code, and just leave essentials relating to the bird I/O, to provide easy reading of the code. More advanced GUIs and output can be added by the end-user. Most of the documentation is included in the script files themselves, but below is a brief guide which will get you started.

To understand the details of the low level I/O, one needs to become familiar with the bird manual. References are made in the documentation to pages from the bird manual, which refer to manual 910007-A Rev A. However, familiarity with the low level I/O is not necessary in use of most of the Bird features. This documentation assumes that the reader is familiar with Matlab.

The units used in the code are meters and radians. Although the output of angles may be printed in degrees.

 

Software Requirements

The attached code has been tested with Matlab version 6.0.0.88 (R12)

The code uses the Instrumentation Toolbox for one command only “instrreset”, which resets the serial ports (calls are made to this function in bird_setup.m). If one did not purchase this toolbox, this line of code will not be run, and the software will work fine as long as one closes all serial ports properly before opening them again. Restarting Matlab will also reset the serial port.

 

Quick Setup

  • Connect all bird cabling and dip switches as shown in manual. The program is currently setup with the Master Bird connected to a serial port, with the other birds in the flock connected to the Master bird via the FBB (Fast Bird Bus). If multiple birds are connected, remember to set the address of the Master to 1 (rather than 0), as described in the manual.
  • Modify file bird_setting_default.m to indicate items such as:
    • COM port (default COM1)
    • Number of birds in flock (default 4)
    • Baud rate (default 115200)
  • For menu driven I/O; from the Matlab command line type
    • run_bird

-         Execute the Setup Bird option the first time run_bird is executed and anytime after Close Serial Port is option executed

-         Select any other option for I/O, plotting, and data display

-         Some of the plots have pull down menus with I/O and plotting options

-         If the Bird panel lights indicate an error, try Reset Bird to correct the problem. You can also select Examine Values to check for Bird error status, and other Bird parameters.

-         If there is a serial port error, try Close Serial Port

-         Exit will bring one back to the command line.

 

  • To get started writing your own Matlab code examine the code in run_bird.m. Some of the key function calls are:
    • bird_setting_default               % loads bird settings into structure “bird”
    • bird = bird_setup(bird);          % setups up serial port and bird settings (bird.bird_port is the serial port object which is returned)
    •  bird_bytes = bird_point_bytes(bird);  % gathers a point of data from a single bird, and returns gathered bytes as bird_bytes
    •  [pos, ang] = bird_bytes_2_data(bird_bytes,bird);   % converts bird_bytes to position and angle values (default units are meters and radians)
    • [R] = bird(ang);          % converts angles to Rotation matrix, where the columns of R are the axes of the bird coordinate systems (note R is the transpose of that used in Bird manual).     
    •  [bird_bytes] = bird_group_bytes(bird);   % will read position of all birds in flock
    • [pos, ang] = bird_group_bytes_2_data(bird_bytes,bird);   % will convert bytes off all birds in flock
    • [pos_ar,ang_ar,s] = bird_group_series(bird); % will repeatedly implement the bird point command to read a series of points on the flock of birds, using a GUI to start and stop the data acquisition
    • [bird_bytes_ar,s] = bird_group_stream(bird); % uses the bird stream command to continuously read data from the flock.
    • [pos_ar,ang_ar,s] = bird_stream_bytes_2_data(bird_bytes_ar,bird,s); % converts data returned from bird_group_stream to position and angle data

 

Key Variables

bird => Structure that contains all bird settings and serial port variables

bird.bird_port => serial port object (get(bird.bird_port) will display port properties)

bird.bird_port.BytesAvailable => number of bytes available in input buffer

bird.examine => bird hardware properties read from bird hardware

bird.group_mode => Matlab flag used to designate if record read is for a dingle bird or group of birds (complete flock)

bird_bytes => record of bytes of data returned from bird as defined in bird manual

pos => position vector of bird sensor. Columns are  x, y, z values in meters; rows are by bird number

ang => angle vector of bird sensor. Columns are thx, thy, thz in radians; rows are by bird number

pos_ar => array of positions in a three dimensional array of the following format

pos_ar(i,X:Z,ibird) => where i is the point counter, X:Z are coordinates, and ibird is the bird counter

ang _ar=> array of positions in a three dimensional array of the following format

ang_ar(i,X:Z,ibird) => where i is the point counter, X:Z are coordinates, and ibird is the bird counter

pos_flock => positions of flock of birds, where each row corresponds to a new bird

pos_flock(X:Z,ibird) => where X:Z are coordinates, and ibird is the bird counter

ang_flock => angles of flock of birds, where each row corresponds to a new bird

ang_flock(X:Z,ibird) => where X:Z are coordinates, and ibird is the bird counter

Angles and Rotation Matrices

The orientation of a bird sensor can be represented by a 3x1 vector of Euler angles, ang, or by a 3x3 rotation matrix,  R. The code is currently written to acquire orientation information in the form of Euler angles to increase I/O speed and to simplify data storage. Once can convert the Euler angles to a Rotation matrix using the function birdR.m.

The Rotation matrix is in the format where each column of R is an axes of the bird coordinate system. The notation in the Bird manual uses the transpose of R.

Matlab provides for easy vector and matrix math to manipulate coordinate system transformations and rigid body rotations. An easy example to follow is in pl_bird_axes.m, where the coordinate axes of each bird sensor are calculated and plotted.

Point vs Stream Data Acquisition

There are two methods for acquiring Bird data continuously. The simplest approach is to implement the Bird Point command repeatedly. This approach will yield a series of positions, but the data acquisition rates will be slow since the bird microprocessor must wait for a host computer command before issuing new data. A series of data points can be acquired in this fashion by using the following command

[pos_ar,ang_ar,s] = bird_group_series(bird)

Faster data acquisition can be achieved by using the Bird Stream command. Once the Stream command is initiated the Bird microprocessor continually sends data to the host computer. This data loads into the serial port buffer, and one needs to ensure that the data from the buffer is removed and stored before the buffer overflows. If the serial port buffer is not read frequently enough then the buffer will overflow. However, if the buffer is read too frequently (such as for every record), then the overhead associated with checking and reading the buffer can slow down the computer, and also cause the buffer to overflow. The solution selected was to read to buffer in “bunches” which are groups of bytes at a time. A stream of data points can be acquired in this fashion by using the following two function calls:

 [bird_bytes_ar,s] = bird_group_stream(bird);

[pos_ar,ang_ar,s] = bird_stream_bytes_2_data(bird_bytes_ar,bird,s);

 

For both methods of continuous data acquisition, a Matlab GUI is used to start and stop the I/O. Matlab does not have a function to monitor the keyboard (such as kbhit in C). Accordingly, the command line does not provide a convenient way to stop a loop with a keyboard input. The GUI approach opens a figure with a pushbutton commands. Implementing the “drawnow” Matlab command within the loop, forces Matlab to monitor the GUI for mouse and button activity.

Trouble Shooting

Matlab Program Hangs

If the program “hangs” the most likely cause is an error with the Bird hardware. Executing a Control C  will exit the Matlab routine, and return one to the command line. One can restart run_bird and then select either Reset Bird or Setup Bird options. If this does not solve the problem, then try unplugging and plugging in the Bird Hardware.

If repeated errors occur one can track them by looking a the LED flashes on the Bird front panel, and looking up the error codes in the Bird manual. One can also execute the bird_examine_value to read the Bird error status.

Currently the Matlab code is implemented in a fashion where the host computer will loop continuously while waiting for the expected number of bytes from the Bird hardware. If these bytes do not arrive, then the computer will hang, which will require the Control C command to break out of the Matlab loop. One could implement a timeout within the  Matlab code to prevent this problem, but one should carefully select a proper timeout value appropriate for the current hardware setup. The timeout feature should be added to all places where bytes are read from the bird.  This code is in the following functions:

bird_point_bytes.m

bird_group_bytes.m

bird_stream_bytes.m

bird_examine_value.m

Serial Port Error

A Serial Port Error can occur if one tries to open a serial port without properly closing the port. Issuing bird_setup.m implements the Matlab Instrument Toolbox command “instrreset” which will reset the serial ports if the toolbox is installed. However, if one does not own the Instrument Toolbox, then try closing the serial port from the menu. If all else fails restarting Matlab will also do the trick.

Erratic Position Output due to Crossing Hemisphere

Erratic position output may be caused by moving from one side of the Bird hemisphere to the other. The Bird hemisphere can easily be set by changing the values in bird_settings_default.m. The Bird hardware is updated in the function  bird_change_value which is called from bird_setup.
